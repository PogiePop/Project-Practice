## 三、application.yml 配置

```yaml
server:
  port: 8080                           # 服务端口

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/project_practice?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root                      # 数据库用户名
    password: your_password             # 数据库密码
    driver-class-name: com.mysql.cj.jdbc.Driver

mybatis:
  mapper-locations: classpath:mapper/*.xml    # XML映射文件路径
  type-aliases-package: com.audit.entity       # 实体类包
  configuration:
    map-underscore-to-camel-case: true         # 下划线转驼峰
```

## 六、接口前缀

所有API路径: `/audit/v1/...`

前端Vite代理配置（vite.config.js）:
```js
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}
```

请求链路: 浏览器 `/api/audit/v1/plan/batches` → Vite代理 → `http://localhost:8080/audit/v1/plan/batches`

## 八、核心业务流程

### 审批-进度联动
```
步骤1(提交)→10%  步骤2(组长)→30%  步骤3(处长)→60%
步骤4(校领导)→85%  步骤5(归档)→100%
驳回→0%+自动生成整改记录
重新提交→重置步骤1
```

### 三向关联传导
```
计划审批完成 → 单位:totalAuditCount+1, latestAuditDate更新
              → 领导:auditCount+1, latestAuditDate更新
              → 领导fundScope = SUM(关联计划projectAmount)
```

### 文件存储
- 上传文件存放: `{workdir}/uploads/`
- 元数据存储: `audit_attachment`表
- 预览: PDF/图片inline, Office文件自动转为下载
