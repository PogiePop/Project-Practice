## 三、application.yml 可配置项详解

```yaml
# ==================== 服务端口 ====================
server:
  port: 8080
  # 说明: HTTP服务监听端口, 默认8080
  # 修改: 如果端口冲突,改为其他值如 9090

# ==================== 数据源 ====================
spring:
  datasource:
    # ---------- 数据库连接 ----------
    url: jdbc:mysql://localhost:3306/project_practice
         ?useUnicode=true              # 启用Unicode支持
         &characterEncoding=UTF-8      # 字符编码(必须UTF-8,否则中文乱码)
         &serverTimezone=Asia/Shanghai  # 时区(中国标准时间)
         &useSSL=false                 # 禁用SSL(本地开发)
         &allowPublicKeyRetrieval=true # 允许客户端获取公钥(MySQL8.0+)
    # ---------- 认证 ----------
    username: root                     # 数据库用户名
    password: your_password            # 数据库密码
    # ---------- 驱动 ----------
    driver-class-name: com.mysql.cj.jdbc.Driver  # MySQL8.0驱动

# ==================== MyBatis ====================
mybatis:
  # ---------- XML映射文件路径 ----------
  mapper-locations: classpath:mapper/*.xml
  # 说明: 扫描resources/mapper/下所有.xml文件
  # 修改: 如需多级目录,改为 classpath:mapper/**/*.xml

  # ---------- 实体类别名 ----------
  type-aliases-package: com.audit.entity
  # 说明: resultType可直接写类名而非全路径

  # ---------- MyBatis配置 ----------
  configuration:
    # 下划线自动转驼峰 (数据库列名 user_name → Java属性 userName)
    map-underscore-to-camel-case: true

    # SQL日志输出 (开发环境建议开启, 生产环境关闭)
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

### 配置项速查表

| 配置项 | 默认值 | 说明 | 何时修改 |
|--------|--------|------|---------|
| `server.port` | 8080 | HTTP端口 | 端口冲突时 |
| `url` | localhost:3306/project_practice | 数据库地址 | 更换数据库服务器 |
| `username` | root | 数据库用户 | 按实际DB账户 |
| `password` | — | 数据库密码 | 按实际密码 |
| `characterEncoding` | UTF-8 | 字符编码 | 永远不要改 |
| `serverTimezone` | Asia/Shanghai | 时区 | 部署在海外时 |
| `useSSL` | false | SSL连接 | 生产环境建议true |
| `mapper-locations` | classpath:mapper/*.xml | XML路径 | XML放在其他目录时 |
| `map-underscore-to-camel-case` | true | 驼峰映射 | 永远不要改 |
| `log-impl` | StdOutImpl | SQL日志 | 生产环境可注释掉 |

## 六、接口前缀

所有API路径: `/audit/v1/...`

前端Vite代理配置（vite.config.js）:
```js
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}
```

请求链路: 浏览器 `/api/audit/v1/plan/batches` → Vite代理 → `http://localhost:8080/audit/v1/plan/batches`

## 八、核心业务流程

### 审批-进度联动
```
步骤1(提交)→10%  步骤2(组长)→30%  步骤3(处长)→60%
步骤4(校领导)→85%  步骤5(归档)→100%
驳回→0%+自动生成整改记录
重新提交→重置步骤1
```

### 三向关联传导
```
计划审批完成 → 单位:totalAuditCount+1, latestAuditDate更新
              → 领导:auditCount+1, latestAuditDate更新
              → 领导fundScope = SUM(关联计划projectAmount)
```

### 文件存储
- 上传文件存放: `{workdir}/uploads/`
- 元数据存储: `audit_attachment`表
- 预览: PDF/图片inline, Office文件自动转为下载
