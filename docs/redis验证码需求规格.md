# 审计信息管理系统 — Redis图形验证码 需求规格说明书

> **文档版本**: V1.0  
> **创建日期**: 2026-07-09

---

## 一、背景

当前登录接口仅校验用户名+密码，缺少人机验证机制，存在暴力破解风险。需引入图形验证码作为登录前置校验，使用Redis缓存验证码并支持一次性消费+自动过期。

## 二、功能需求

| 编号 | 需求 | 优先级 |
|------|------|--------|
| CAP-01 | 生成4位数字+字母混合验证码图片（含干扰线+噪点），返回Base64 | P0 |
| CAP-02 | 验证码存入Redis，key=`captcha:{uuid}`，value=验证码文本，TTL=120s | P0 |
| CAP-03 | 登录接口新增captchaUuid、captchaCode参数，前置验证码校验 | P0 |
| CAP-04 | 验证码校验通过后立即删除Redis缓存（一次性消费） | P0 |
| CAP-05 | 验证码不区分大小写 | P1 |
| CAP-06 | 验证码过期/错误返回区分错误码和文案 | P0 |
| CAP-07 | 前端登录页新增验证码输入框+图片组件，点击刷新 | P0 |

## 三、接口清单

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| GET | `/audit/v1/captcha/generate?uuid=xxx` | 生成验证码图片Base64 | 白名单 |
| POST | `/audit/v1/auth/login` | 登录（新增captchaUuid+captchaCode） | 白名单 |

## 四、Redis缓存规范

| 项 | 值 |
|-----|-----|
| key格式 | `captcha:{uuid}` |
| value | 4位验证码文本（如"AB3K"） |
| TTL | 120秒 |
| 序列化 | StringRedisSerializer（避免\xAC\xED乱码） |

## 五、错误码

| code | 说明 |
|------|------|
| 1006 | 验证码不存在或已过期 |
| 1007 | 验证码输入错误 |

## 六、验收标准

- [ ] GET `/audit/v1/captcha/generate?uuid=abc123` 返回Base64图片
- [ ] POST `/audit/v1/auth/login` 不带验证码参数 → 1006
- [ ] 输入错误验证码 → 1007
- [ ] 同一验证码使用两次 → 第二次1006（已销毁）
- [ ] 120秒后验证码自动失效 → 1006
- [ ] 前端登录页显示验证码图片，点击刷新换图
- [ ] 原有JWT、RBAC功能不受影响
