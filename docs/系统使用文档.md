# 审计信息管理系统 — 使用文档

## 一、系统概述

审计信息管理系统是一套面向高校审计处的信息化管理平台，覆盖审计计划、审计对象、审计进度三大核心业务，实现"计划-项目-成果"全流程数字化管理。

### 技术架构

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + Element Plus + Vite + Axios |
| 后端 | Spring Boot 3.4 + MyBatis + MySQL |
| 数据库 | MySQL 8.0（project_practice，utf8mb4） |

---

## 二、环境部署

### 2.1 前置条件

- Node.js ≥ 18
- JDK ≥ 17
- MySQL 8.0
- Maven 3.9+

### 2.2 数据库初始化

```bash
mysql -uroot -p < sql/init.sql
```

初始化后包含 7 张核心表 + 种子数据（计划/单位/干部/审批/整改/消息/用户）。

### 2.3 后端启动

```bash
cd backend
# 修改 src/main/resources/application.yml 中的数据库连接信息
mvn spring-boot:run
```

后端运行在 `http://localhost:8080`，接口前缀 `/audit/v1`。

### 2.4 前端启动

```bash
cd front
npm install
npm run dev
```

前端运行在 `http://localhost:5173`，Vite 代理 `/api` 到后端 `http://localhost:8080`。

## 三、功能模块

### 3.1 登录

访问 `http://localhost:5173/login`，输入用户名和密码。登录后 Token 存储在 localStorage。

### 3.2 审计计划管理

**路径：** `/plan/form`

| 功能 | 操作 |
|------|------|
| 查看计划列表 | 支持按名称/类型/年度/状态筛选 |
| 新建计划 | 点击"新建计划批次"，填写编号/名称/类型/周期/组长/被审单位/经责干部 |
| 编辑计划 | 点击行内"编辑"按钮 |
| 删除计划 | 点击行内"删除"按钮（仅草稿可删） |
| 穿透查询 | 点击"穿透"查看计划完整信息+附件+整改台账 |
| 附件管理 | 点击"附件"上传/预览/删除附件（支持 PDF/Word/Excel/图片） |
| 方案模板 | 点击"方案"绑定标准化审计方案模板 |
| 智能推荐 | 点击"推荐"自动筛选应审计的单位/干部 |
| 计划变更 | 点击"变更"修改计划信息，提交后在审批跟踪中确认生效 |
| 导入/导出 | 表头按钮：批量导入Excel / 导出Excel |

### 3.3 审批跟踪

**路径：** `/plan/track`

| 功能 | 操作 |
|------|------|
| 选择计划 | 下拉选择要跟踪的计划批次 |
| 审批进度 | 查看步骤条 + 审批意见时间线 |
| 审批操作 | 填写意见 → 审批通过 / 驳回 |
| 被驳回 | 显示红色警告 + 自动生成整改记录 → 点击"重新提交" |
| 审批历史 | 查看历史审批记录 |
| 变更记录 | 查看计划变更记录 → 点击"确认"使变更生效 |

**审批-进度联动：**
- 步骤1(提交)完成→10% → 步骤2(组长)30% → 步骤3(处长)60% → 步骤4(校领导)85% → 步骤5(归档)100%
- 驳回→0% + 自动生成整改

### 3.4 被审计单位库

**路径：** `/object/unit`

| 功能 | 操作 |
|------|------|
| 列表查询 | 按名称/分类筛选 |
| 新建单位 | 填写单位信息，分管校领导/财务联系人从干部库选择 |
| 编辑/删除 | 行内操作按钮 |
| 详情 | 查看审计记录+整改台账+专项资金 |
| 整改台账 | 查看该单位所有未完成整改项 |
| 导入/导出 | Excel 批量导入（12列）/ 导出（13列含审计信息） |

### 3.5 经责领导干部库

**路径：** `/object/lead`

| 功能 | 操作 |
|------|------|
| 列表查询 | 按姓名/工号/在职状态筛选 |
| 新建干部 | 填写信息（任职年限自动计算） |
| 编辑/删除 | 行内操作按钮 |
| 详情 | 查看基本信息+任职履历时间线+关联审计项目 |
| 任职履历 | 点击"履历"→添加/编辑/删除任职记录，支持导入导出 |
| 滚动推荐 | 点击"滚动推荐"查看应纳入经责审计的干部清单 |
| 导入/导出 | Excel 批量导入（9列）/ 导出（13列） |

### 3.6 审计进度可视化

**路径：** `/statistic`

展示年度计划执行情况的全局视图：统计卡片、按类型柱状图、甘特图、审计组长负载、预警项目。

### 3.7 个人信息与系统设置

| 路径 | 功能 |
|------|------|
| `/settings/profile` | 查看/编辑个人信息，修改密码 |
| `/settings/system` | 通知偏好、界面显示、关于系统 |

## 四、数据关联说明

```
审计计划（audit_plan_batch）
  ├── unit_id → 被审计单位（audit_object_unit）
  │     ├── totalAuditCount（审批完成+1）
  │     ├── latestAuditDate（审批完成更新）
  │     └── pendingRectifyCount（从整改台账实时统计）
  ├── leader_id → 领导干部（audit_object_leader）
  │     ├── auditCount（审批完成+1）
  │     ├── latestAuditDate（审批完成更新）
  │     └── pendingRectifyCount（级联自单位）
  └── 整改台账（audit_rectify_ledger）
        ├── 审批驳回 → 自动生成（类别=审批驳回）
        └── 审批通过 → 自动清除驳回类整改
```

## 六、常见问题

**Q: 前端请求报 404？**
A: 确认后端已启动在 8080 端口，Vite 代理配置正确。

**Q: 数据库连接失败？**
A: 检查 `application.yml` 中的数据库用户名/密码/端口。

**Q: Excel 预览乱码？**
A: Office 文件不支持浏览器内联预览，会自动触发下载。

**Q: 修改任职起始后年限不变？**
A: 年限在查询/创建/更新时动态计算 `(today - startDate) / 365`，刷新列表即可看到。
